# HTTPS Server (Secure Web Server with CGI)

Веб-сервер с поддержкой HTTPS/TLS, написанный на чистом C с использованием ассемблера для оптимизации критических участков. Поддерживает CGI, CORS, песочницу (chroot) и многопоточную обработку запросов.

## 🛠 Технологический стек

### Backend (Серверная часть)
* **Язык:** C (стандарт C99, POSIX)
* **Оптимизация:** GNU Assembler (x86-64) для функций сетевого ввода-вывода и парсинга заголовков
* **Криптография:** mbedTLS (TLS 1.2, SSL handshake)
* **Многопоточность:** POSIX Threads (pthreads) — пул потоков для обработки соединений
* **Безопасность:**
  * Изоляция процессов (chroot)
  * Понижение привилегий (запуск от имени пользователя `nobody` или указанного в конфиге)
  * Защита от DOS (rate limiting логирования и запросов)
  * Валидация путей (защита от Path Traversal)

### Frontend (Веб-интерфейс)
* **Языки:** HTML5, CSS3, Vanilla JavaScript (без тяжелых фреймворков)
* **Стили:** Material Design Icons, кастомные CSS-анимации
* **Функционал:**
  * Панель управления конфигурацией сервера
  * Стресс-тестирование (нагрузочное тестирование)
  * Демонстрационные страницы (калькулятор, CSS-анимации)

## 📋 Зависимости

Для сборки и запуска необходимы следующие пакеты (на примере Debian/Ubuntu):

```bash
sudo apt-get update
sudo apt-get install clang make libmbedtls-dev libmbedcrypto-dev libmbedx509-dev openssl
```

* **Компилятор:** `clang`
* **Ассемблер:** `as` (GNU as)
* **Библиотеки:** `mbedtls`, `pthread`
* **ОС:** FreeBSD, Linux (требуется для системных вызовов `chroot`, `setuid`, `fork`)

## 🚀 Сборка

1. **Генерация сертификатов (если отсутствуют):**
   Сервер требует наличия `cert.pem` и `key.pem`. Для тестов можно создать самоподписанный сертификат:
   ```bash
   openssl req -x509 -newkey rsa:2048 -keyout key.pem -out cert.pem -days 365 -nodes
   ```

2. **Компиляция:**
   ```bash
   # Сборка релизной версии (оптимизированная)
   make release

   # Или сборка отладочной версии
   make debug

   # Очистка артефактов
   make clean
   ```

   В результате будут созданы:
   * `server` — бинарный файл сервера
   * `cgi/config.cgi` — CGI-скрипт для управления конфигурацией

---

## ⚙️ Запуск

1. **Настройка:**
   Отредактируйте `server.conf` при необходимости (порт, пути к сертификатам, настройки CGI).
   * По умолчанию порт: `8443`
   * Корневая директория веб-сайта: `./www`

2. **Запуск сервера:**
   ```bash
   # Запуск с конфигурацией по умолчанию
   ./server

   # Запуск с указанием файла конфигурации
   ./server my_config.conf
   ```

   > **Примечание:** Для использования портов ниже 1024 или функции `chroot` могут потребоваться права root. Сервер автоматически понизит привилегии после запуска в соответствии с настройками `user` в конфиге.

3. **Доступ:**
   Откройте браузер и перейдите по адресу: `https://localhost:8443`

## 🔌 API Endpoints

* `GET /` — Главная страница
* `GET /settings` — Панель управления настройками
* `GET /cgi-bin/config.cgi?action=read` — Получить текущую конфигурацию (JSON)
* `POST /cgi-bin/config.cgi` — Обновить конфигурацию (action=write)
* `GET /tools/stress-test.html` — Инструмент нагрузочного тестирования

## 🛡 Особенности безопасности

1. **TLS 1.2:** Все соединения зашифрованы
2. **Sandboxing:** Поддержка `chroot` для изоляции файловой системы
3. **Privilege Separation:** Возможность запуска от имени непривилегированного пользователя
4. **Input Validation:** Защита от обхода путей (`..`) и переполнения буфера
5. **CORS:** Гибкая настройка Cross-Origin Resource Sharing
